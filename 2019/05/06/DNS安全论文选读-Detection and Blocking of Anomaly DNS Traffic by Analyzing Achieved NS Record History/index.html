Detection and Blocking of Anomaly DNS Traffic by Analyzing Achieved NS Record History

来源

Ichise H, Jin Y, Iida K, et al. Detection and Blocking of Anomaly DNS Traffic by Analyzing Achieved NS Record History[C]//2018 Asia-Pacific Signal and Information Processing Association Annual Summit and Conference (APSIPA ASC). IEEE, 2018: 1586-1590.

关键思想

名称解析会获得之前发送解析请求的NS记录和A记录，而被僵尸程序感染的机器会直接发送带外查询而不会获得NS和A记录。

在组织内部机器通常会经过设置的DNS服务器，被感染主机不会经过内部设置的DNS服务器，直接发送解析请求？

三种基于DNS的Botnet通信

  • via-resolver DNS query

    依赖于内部DNS服务器，利用DNS查询中的TXT记录进行通信

  • indirect outbound DNS query

    先通过内部DNS服务器查询获得C2服务器的IP地址，然后向C2服务器进行DNS查询

  • direct outbound DNS query

    C2服务器的地址被硬编码在僵尸程序中，可以直接向C2服务器发起查询

系统介绍

  1. NS记录和胶水A记录白名单

  2. 使用SDN技术实现对DNS包的控制

构建NS历史记录数据库

收集内部DNS服务器的流量，获取合法的NS记录数据

系统结构

  1. 用户通过DNS服务器发送解析请求，用户所有的DNS请求包都被openflow 交换机监测。
  2. 如果流表中没有DNS请求包中的目的IP地址，则openflow 交换机利用openflow 控制器检查目的IP地址
  3. openflow 控制器针对目的IP地址查询数据库
  4. 数据库回复查询响应
  5. openflow 控制器根据回应做出决策。如果目的IP地址在数据中，则允许请求包通过，否则将其丢弃。
