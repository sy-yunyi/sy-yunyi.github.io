Commando VM: 第一个发行版Windows攻击平台

对于寻找稳定且受支持的Linux测试平台的渗透测试人员来说，业界一致认为Kali是首选平台。但是如果使用Windows操作系统，你会发现并没有这样一个令人满意的平台。对于安全研究人员来说，我们每个人可能都至少需要花几个小时定制一个Windows工作环境，即使我们在客户要求中使用相同的工具、程序和技术。因此，在保持所有工具集都是最新的同时维护自定义环境对所有人来说都是单调乏味的工作。认识到这些，我们创建了一个Windows发行版，主要支持渗透测试人员和红队。

诞生于我们著名的专注于逆向工程和恶意软件分析的FLARE VM中，完整的Mandiant攻击环境(“Commando VM”)附带自动化脚本，帮助每个人构建自己的渗透测试环境，并简化平台配置和部署的过程。这篇博客目的在于讨论Commando VM的特性、安装说明以及介绍平台的一个使用用例。

关于Commando VM

渗透测试人员通常使用他们自己的定制的Windows机器评估活动目录（Active Directory）环境。Commando VM是专门为执行这些内部渗透测试而设计的平台。使用Windows机器的好处包括，Windows和活动目录（Active Directory）的原生支持、使用虚拟机作为C2框架的临时区域、浏览分享更容易和更具有互动性及不用再担心在使用PowerView和BloodHound等工具时在客户端上输出文件。

Commando VM使用Boxstarter、Chocolatey和MyGet等软件包来安装所有软件，并提供了许多工具和实用程序来支持渗透测试。

工具数目高达140多个，其中包括:

  • Nmap
  • Wireshark
  • Covenant
  • Python
  • Go
  • Remote Server Administration Tools
  • Sysinternals
  • Mimikatz
  • Burp-Suite
  • x64dbg
  • Hashcat

具备这么多功能的Commando VM的目标是能够成为真正帮助每一个渗透测试工作者和红队的Windows机器。看到这篇文章的蓝队也不用担心，我们也完全的同样支持蓝队。Commando VM中包含的多功能工具集为蓝团队提供了必要的工具来审计他们的网络并提高他们的检测能力。在同样的攻击工具库里，蓝队就可以很容易地了解攻击工具和趋势。

安装

类似于FLARE VM，我们推荐在虚拟机中使用Commando VM。这可以简化部署，并可以利用快照功能在每次使用之前恢复到原始状态。我们假设你具有设置和配置自己的虚拟化环境的经验。首先创建一个新的虚拟机(VM)，最低的要求是:

  • 60G的硬盘空间
  • 2G内存

接下来安装Windows。Commando VM被设计安装在Windows 7 SP1或Windows 10上，Windows 10可以使用更多功能。

Windows安装完成后，建议安装特定的VM客户工具(例如VMware Tools)，以允许复制/粘贴和调整屏幕大小等附加功能。

  1. 使用Windows Update确保Windows已完全使用最新补丁更新。注意:重新启动后可能需要再次检查更新。
  2. 建议在安装之前对VM进行快照，以获得一个纯净的Windows实例。
  3. 进入下面的网址，下载Commando VM到虚拟机
    • https://github.com/fireeye/commando-vm
  4. 按照以下步骤完成Commando VM的安装:
    1. 将Commando VM解压到你选择的目录。
    2. 打开管理员模式的PowerShell的新会话，因为Commando VM会试图安装额外的软件和修改系统设置。
    3. 在PowerShell中刚进入Commando VM所在的目录。
    4. 通过执行以下命令将PowerShell的执行策略更改为无限制，在PowerShell提示时回答“Y”。
      • Set-ExecutionPolicy unrestricted
    5. 执行install.ps1安装脚本。系统将提示输入当前用户的密码。Commando VM需要当前用户的密码才能在重启后自动登录。你可以选择在命令行上使用“-password ”来指定当前用户的密码。

图2 正在运行的安装脚本

剩下的安装步骤全部都是自动的。根据网络状况，整个安装可能需要2到3个小时来完成(国内需要的时间可能更长，还可能遇到下载失败的情况，可以进行多次尝试)。由于大量的软件安装需求，VM将重启多次。安装完成后，PowerShell提示符将保持打开状态，等待你按下任意键退出。

安装完成后，桌面环境如下：

图3 安装后的桌面环境

此时，建议重新启动计算机，以确保最终的配置更改生效。

重启后，Commando VM安装完成!

建议将此时环境建立另一个快照，以保存一个纯净的VM状态，以便在未来使用。

概念验证

构建Commando VM的主要目的是支持内部活动。为了展示Commando VMs功能，我们构建了一个活动目（Active Directory）部署示例。这个环境是人为设置的，但也展示了红队在实际环境中经常观察到的错误配置。

首先寻找容易查找的漏洞点，我们发现一台主机在TCP端口8080上运行一个有趣的web服务器，该端口通常用于管理目的。使用Firefox，我们可以通过TCP端口8080通过HTTP连接到服务器。

结语

演示中使用的所有工具都默认安装在Commando VM上，当然还有很多其他工具。有关完整的工具列表和安装脚本，请参见Commando VM Github repo。我们期待获取和处理用户的反馈，添加更多的工具和特性，并创建许多增强功能。我们相信这个发行版将成为渗透测试人员的标准工具，并期待Windows攻击平台的持续改进和开发。
