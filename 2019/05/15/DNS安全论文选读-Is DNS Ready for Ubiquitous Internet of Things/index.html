Six's Blog
2019-05-15
论文笔记
DNS, IOT, 安全

Is DNS Ready for Ubiquitous Internet of Things

来源

IEEE Access 7: 28835-28846 (2019) Zhiwei Yan, Hongtao Li, Sherali Zeadally, Yu Zeng, Guanggang Geng:

综述

IOT的意愿不仅仅是覆盖具有完整规则的特定应用，还应该涵盖物理世界中更通用对象或事物或设备的普遍连接和虚拟世界中的相关信息。

  • 智慧城市

    1. 智慧城市交通网络
    2. 升级的饮水供给网络
    3. 垃圾处理设施
    4. 更加有效为建筑照明和供暖

为了实现这些需要对每一个对象进行独特的名称解析，需要一个安全的、可扩展的、高效的名称解析。

目前有两大类解决方案

  • 基于现有DNS
  • 网络演变出的新设计的规则

本文要研究的问题

现有的DNS所提供的名称解析服务可以充分的满足未来的IOT需求么？

研究方法

分析现有DNS在支持IOT时的能力和挑战。

从五个特征考虑：

  • 安全（security）
  • 移动性（mobility）
  • 基础设施的独立性（infrastructure independence）
  • 地区性（localization）
  • 高效性（efficiency）

简称为SMILE

本文贡献

  • 分析典型的IOT设备及其对名称解析的关键性要求 - SMILE
  • 分析DNS的功能和性能，分析其部署模型在SMILE下的缺点和优点
  • 总结在不同场景下的基于DNS的扩展和全新的设计的名称解析服务，揭示未来IOT名称解析服务的发展

DNS的演变与发展

从域名的两种特性：商标和协议标识两部分，介绍域名的含义和商业性价值。

介绍域名的注册流程，及相关管理机构。

域名空间的扩展，至2012年有1200多个新的顶级域开始使用。

DNS记录角度，现在存在着众多的记录类型以提供不同的服务，但是只有某几种类型的记录使用频率非常高。

DNSSEC带来的变化和影响。

IOT名称服务的要求-SMILE

Security

在物联网中，设备之间相互连接，提高了被攻击的概率。安全是物联网最基础的赋能者。

名称服务安全要求

足够鲁棒、安全、可信

鲁棒性代表抵抗攻击的能力，需要有足够的扩展能力，并且其提供的服务不依赖其他系统。

支持数据验证，阻止欺骗和篡改数据且保护关联名称的隐私，应用现有成熟的证书技术。

示例

Named Data Networking(NDN)重新设计了基于TCP/IP的通信模型。

Mobility

思科的Visual Networking Index，预计在2020年全球的移动用户将达到55亿，占全球人口的70%。

98%的移动数据由不同的智能设备产生。

在物联网之前，移动性就是IP所面临的挑战。

对于传统解决移动性的方案对物联网并不可行，因为物联网设备资源有限，数量庞大且可能造成局部最优现象。

物联网对移动性的要求之高会给网络演变带来新的挑战。

示例

MobilityFirst 通过将网络设备与名称分离提高移动性，将Global Unique IDentification作为其命名策略。

Global Unique IDentification 的设计是基于设备和标识符的分离规则

NDN同样提供了提供移动性的方案，但是难以支持发行商。

Infrastructure Independence

对于物联网，其网络拓扑是动态变化的且会引进资源相关或者功能相关的智能设备。

zero-configuration networking可以支持设备自主的配置网络地址，名字和发现服务。但是名称手动配置问题随着设备的增多而加剧。

名称解析同时还要满足不依赖于基础设施，防止提供名称解析的基础设施出现故障。

示例

oneM2M采用Object Identifier,名称由管理等级信息和设备等级信息组成，具有良好的可识别性。

Localization

物联网设备的区域性包括两个部分，工作范围区域性和使用习惯区域性。

对于工作范围区域性，在特性情景下的设备可能仅仅在某个局域网内进行工作，不需要全局的地址。对于使用习惯区域性，针对不同地区的用户物联网设备应该匹配其使用习惯。

名称服务要求

  • 不同的地区其名字是可用并且可理解的
  • IOT设备中使用的名称标签对本地用户友好，尊重本地使用习惯
Efficiency

对于时延敏感的IOT设备效率是非常重要的。

随着技术的发展，各方面对效率的要求越来越高。

IOT设备的名称配置和解析必须具有高效的支持实时和时延敏感的应用，以避免影响IOT设备的整体性能。

示例

PUSRSUIT(Publish-Subscribe Internet Technology)是一种在内容中心概念下的支持内容发布与订阅的新的体系结构。管理的核心元素是集结系统和拓扑管理。

在SMILE下DNS的适应与挑战

Security

为了应对DNS所面临的问题，IETF提出了DNSSEC。DNSSEC使用非对称的加密算法来保证数据不被篡改。

但是DNSSEC引进了更多的记录，为解析的性能带了较大的影响，同时，并没有解决信息泄露的问题。

针对信息泄露，可以使用TLS，但同样会使得性能下降。

在IOT的背景下，DNS需要更加轻量级的安全模式。

Mobility

对于DNS中的更新协议，当信息发生改变时，数据可能不会立刻更新，但是对于IOT设备来说，如果地址或者名称发生改变，必须尽可能快的更新信息。

DNS对移动性的支持的另一个限制因素是递归服务器的缓存，移动节点的变化需要将TTL设置为0。

虽然在DNS中也有一些移动解决方案，如MIPv6，PMIPv6。但是其并没有直接将DNS作为管理IOT设备的名称地址映射的数据库，而是将其用作一个定位管理设备信息数据库的发现服务。

Infrastructure Independence

利用Neighbor Discovery Protocol(NDP)和Dynamic Host Configuration Protocol(DHCP)可以实现利用基础参数对设备进行初始化配置。

在没有有效的DNS基础设施的地区为了扩展名称服务，利用基于Over Multicast DNS(mDNS)的DNS-based Service Discovery(DNS-SD)

Localization

通过增强区域性的的递归解析和权威服务器，来提高DNS的性能。

在基于区域的服务中需要地理信息，存在两种实现方式：在实体配置名称时，将名称与地理信息进行绑定；DNS协议支持维持位置相关的记录LOC RR

在区域性习惯方面，DNS已经支持国际化域名。

Efficiency

DNS是层次结构，在解析时需要层层查询，需要的延时可能较长。

通过实验，表明随着缓存无法命中的的增加，延时逐渐升高。

DNS利用本地缓存来降低解析延时，利用任播缩短服务距离，但是仍无法有效的满足IOT对高效，高扩展性的要求。

总结

IPv6的地址空间可以支持IOT设备的地址要求，但是如何满足其对可扩展性、可靠性、高效性名称服务的要求，仍然是挑战。

分析

  1. 虽然DNS增强了安全，但是代价太高，不适合IOT设备
  2. 虽然DNS支持动态更新，但是其设计目的并不是面向移动性的，而且名称更新很复杂
  3. DNS解析没有本地设施的依赖性，可以很好的适应本地连接的的扩展，和一些新出现的技术，如边缘计算
  4. DNS在服务部署和名称格式的区域性上在发展
  5. 由于层次结构和不可预知的缓存导致的时延，使得高效性仍是一个巨大的挑战
