路由基本概念

路由

指通过相互连接的网络吧信息兄源结点传输到目标结点的活动。一般而言，在路由过程中，信息至少会经过一个或多个中间结点。路由交换发生在OSI参考模型的第三层-网络层。

路由技术的关键点是如何选择某条最佳路径将信息送到目标结点。

路由器

路由器是互联网的主要结点设备，通过运行路由算法决定数据的转发。转发策略称为路由选择。

路由作为不同不同网络之间相互连接的枢纽，构成了基于TCP/IP的国际互联网络的主题脉络，其处理速度是网络通信的主要瓶颈之一，其可靠性则直接影响这网络互连的质量。

路由表

路由表记录着到达不同的网络的路线。一个路由信息包括：

  • 目的网络，用以匹配报文的目的地址进行路由选择
  • 下一跳地址，指明路由的发送路径
  • 跳跃数，标识路径的好坏，是进行路由选择的标准

网络层协议根据目的地址信息查找到响应的路由信息，将报文按照正确的路径转发出去。

路由器功能

协议转换

不同的路由器有不同的路由器协议，一般路由器会支持多种网络协议，实现不同协议、不同体系结构网络之间的互连互通。

寻址

路由器中的寻址动作与主机类似，区别在于路由器不止一个出口，所以不能通过简单的配置一个默认网关解决所有数据分组转发，必须根据目的网络的不同选择对应的出口路径。

分组转发

路由器的主要功能是分组转发，在路由协议的支持下，路由器根据分组的目的地址将数据包从最合适的端口转发出去，从而实现互连互通。

路由分类

典型的路由选择方式有两种：静态路由和动态路由。

静态路由在路由器中设置固定的路由表，由管理员负责创建和维护。

动态路由是网络中的路由器之间相互通信、传递路由信息、利用收集的路由信息更新路由表。可以实现实时的适应网络结构的变化。动态路由按照区域（自治系统）划分，可分为内部网关协议（Interior Gateway Protocol,IGP）和外部网关协议（Exterior Gateway Protocol, EGP）两大类。

动态路由网关协议分类：

IGP

  • 链路状态 - OSPF
  • 距离矢量 - RIPv1、RIPv2、IGRP
  • 混合型 - EIGRP、DEIGRP

EGP

  • EGP
  • BGP
路由器端口配置原则
  1. 路由器的物理网络端口通常有一个IP地址
  2. 相邻路由器的相邻端口的IP地址必须在同一IP网段上
  3. 同一路由器的不同端口的IP地址必须在不同的IP网段

静态路由

静态路由是在路由器中手动配置的固定路由，路由明确的指定了数据包到达目的地必须经过的路径，除非人工干预，否则静态路由不会自动改变。

静态路由不会队网络的改变做出反应，所以静态路由适用于网络规模不大、拓扑结果相对固定的网络环境。

一条静态路由的条目通常由三个部分组成：目的IP地址对应的子网、子网掩码、网关。

静态路由配置
  1. 为路由器每个端口配置IP地址
  2. 确定该路由器有哪些直连网络的路由信息
  3. 确定网络中有哪些非直连网段
  4. 添加该路由器的非直连网段的相关信息
