渗透技术基础之DVWA中XSS漏洞利用与源码分析

发表于 2019-06-04 | 分类于 安全笔记 | |
字数统计: 2.3k | 阅读时长 ≈ 11

本文基于DVWA，从源码角度分析XSS漏洞的挖掘及防御措施。

DVWA (Dam Vulnerable Web Application)DVWA是用PHP+Mysql编写的一套用于常规WEB漏洞教学和检测的WEB脆弱性测试程序。包含了SQL注入、XSS、盲注等常见的一些安全漏洞。

下载地址：http://www.dvwa.co.uk/

安装可以采用wamp，过程较简单在这不再介绍。

DVWA中包含多种漏洞，这里仅仅对XSS进行分析。

DVW中的XSS分为三类，反射型、存储型和DOM型。关于几种类型的详细介绍可以看之前的文章。

反射型XSS

DVWA中的漏洞已经做了等级划分，分为low、medium、high和impossible。

low等级基本是不做任何防护，而medium、high和impossible分别做了不同等级的防护。

low等级

low等级中代码非常简单，没有对输入参数做任何防护，所以直接可以实现XSS。

输入<script>alert(1)</script>

medium等级

可以看到，medium等级中做了str_replace的过滤，但是过滤仅仅针对于<script>，采用大小写就可以很简单的进行绕过。

输入<sCript>alert("XSS")</script>

high等级

high等级中利用preg_replace对<script>做了正则匹配，这里就不能再利用script标签，我们可以换其他标签，如img.

输入<img src=# onerror=alert("xss")>

impossible等级

impossible等级中采用了htmlspecialchars进行过滤。htmlspecialchars把预定义的字符转换为 HTML 实体，阻止了浏览器对恶意代码的解析执行，是目前比较好的防止XSS的方法，但是有时也不能完全的防御，如其默认配置是不过滤单引号的，就可以利用单引号导致提前闭合，然后执行其他的恶意代码。

为了实现盗取cookie的目的，首先在机器上建立了一个数据库dvwacookie,创建一张表cookie，表中包含一个字段cooke。

存储型XSS

low等级

同样在low等级中，没有做任何防护。

在 Message 中输入 <script>alert("xss")</script>

medium等级

可以看到在medium等级中，对message的过滤是比较完善的，使用了htmlspecialchars函数，基本上很难绕过。但是对于name仅仅用了str_replace，可以尝试利用大小写过滤。

在输入发现页面对name的输入长度做了限制，但是仅仅是在前端做了限制，最简单的方法就是查看源码将长度改掉，或者你也可以进行抓包修改数据。

在name输入<Script>alert("xss")</script>,可以实现XSS。

high等级

和反射中的high类似，使用了正则表达式进行过滤，同样可以利用其他标签绕过。

注入点同样在name，输入<img src=# onerror="alert('xss')">，实现XSS。

impossible等级

对message和name都利用htmlspecialchars进行过滤。
