字符串移位包含问题

Six's Blog
2019-12-15
算法
字符串, 编程之美

问题描述

给定两个字符串s1和s2，判定s2是否能够被通过s1作循环移位得到字符串包含。

例子

给定 s1=AABCD，s2=CDAA，则返回 true 。

给定 s1=ABCD，s2=ACBD，则返回 false 。

分析

解法1

最简单的方法是对s1进行循环移位，然后判断s2是否在其中，遍历所有移位的可能性。

书中代码这里有问题，条件判断写错了。

strstr() 函数声明

  • char strstr(const char haystack, const char *needle);
  • 该函数返回在 haystack 中第一次出现 needle 字符串的位置，如果未找到则返回 null。
  • 头文件 ：#include <string.h>
  • note：如果没有找到则返回的是 NULL ，这里用 0 进行比较，也可以用 bool 类型的数据接收返回值，如果找到则为 true ，否则为 false 。原本函数的返回值为 char * 类型，这里作了隐式的转换。

python实现

但利用穷举法作移位，如果字符串长度很长，则效率很低。

解法2

空间换时间

再来看题目。题目要求是判断s2是否存在于s1移位后的字符串中，那么我们来看一下，移位会对s1有什么影响。

假定s1为：ABCD

ABCD -> BCDA ->CDAB -> DABC -> ABCD

如果我们将前面移走的数据保留下来，则有

ABCD -> ABCDA -> ABCDAB -> ABCDABC -> ABCDABCD

所以，s1移位得到的字符串都是s1s1的字串，那么如果s2在s1s1上，则s2肯定在s1移位的字符串上。

所以题目可以转换为：s2是否在字符串s1s1上。这样就不用作循环移位了。

C/C++实现

malloc()

函数原型：void *malloc(unsigned int num_bytes);

参数：num_bytes 是无符号整型，用于表示分配的字节数。

返回值：如果分配成功则返回指向被分配内存的指针(此存储区中的初始值不确定)，否则返回空指针NULL。

功能：分配长度为num_bytes字节的内存块

头文件：# include<malloc.h>

strcpy()

函数原型：char strcpy (char dest, const char* src);

参数：des为目标字符串，source为原字符串。

返回值：目标串的地址。

头文件：#include <string.h>

功能：把从src地址开始且含有 \0 结束符的字符串复制到以dest开始的地址空间，返回值的类型为char*。

strcat()

函数原型：char strcat(char dest, const char *src);

头文件：#include <string.h>

返回值：dest 字符串起始地址。

功能：

1、dest 为目的字符串指针，src 为源字符串指针。strcat() 会将参数 src 字符串复制到参数 dest 所指的字符串尾部；

2、dest 最后的结束字符 NULL 会被覆盖掉，并在连接后的字符串的尾部再增加一个 NULL。
