ping命令

ping（Packet Internet Grope，因特网包探测器），在进行网络实验、调试过程中，ping是最常用的命令。一般用来测试源主机与目的主机网络的连通性。

ping的作用范围是IP层，利用回应请求/应答ICMP报文来测试目的主机或路由器的可达性。

ping命令可获得的信息：

  • 检测网络的连通性，检验与远程计算机或本地计算机的连接。
  • 确定是否由数据报被丢失、复制或重传。
  • 计算数据包往返时间（Round Trip Time,RTT）
  • 确定数据包是否被破坏

ping命令原理

向指定的网络地址发送一定长度的数据包，按照约定，若指定网络地址存在的话，会返回同样大小的数据包，当然，若在特定时间内没有返回，就是“超时”，会被认为指定的网络地址不存在。

  2. 设置生存时间（Time to Live，TTL）

    TTL指一个网络层的数据包的生存周期。不同的操作系统对TTL赋予不同的默认值，可以通过某些系统的网络参数来改变，如win默认为128，可以通过注册表进行修改；而Linux默认为64。

    TTL的作用是在含有长路径或者环路情况下，令设备抛弃ICMP Request包。原因在于如果传输路径很长，说明路径非常复杂或者可能存在环路。如果存在环路，不进行终止的会影响网络的正常运行。利用TTL，可以实现将数据包控制在某个路径长度的范围。在传输过程中，每经过一个结点，将TTL值减1，若TTL值为0，则这个数据包将会被抛弃，无论是否到达终点。

  6. 其他参数

    • -n count：win 定义用来测量所发出的测试包的个数，默认值为4。
    • -c count：linux 指定要被发送（或接收）的回送信号请求的数目，由 Count 变量指出。
    • -f : linux 指定 flood-ping 选项。 -f 标志“倾倒”或输出信息包，在它们回来时或每秒 100 次，选择较快一个。每一次发送 ECHO_REQUEST，都打印一个句号，而每接收到一个 ECHO_REPLY 信号，就打印一个退格。这就提供了一种对多少信息包被丢弃的信息的快速显示。仅仅 root 用户可以使用这个选项。
    • -v ：win tos 将“服务类型”字段设置为 tos 指定的值。
    • -v : linux 请求详细输出，其中列出了除回送信号响应外接收到的 ICMP 信息。
    • -r ：win 在“记录路由”字段中记录传出和返回数据包的路由。通常情况下，发送的数据包是通过一系列路由才到达目标地址的，通过此参数可以设定，想探测经过路由的个数。限定能跟踪到9个路由。
    • -r : linux 忽略路由表直接送到连接的网络上的主机上。如果 主机 不在一个直接连接的网络上，ping 命令将产生一个错误消息。这个选项可以被用来通过一个不再有路由经过的接口去 ping 一个本地主机。
    • -s ：win 指定 count 指定的跃点数的时间戳。与参数-r差不多，但此参数不记录数据包返回所经过的路由，最多只记录4个。
    • -s PacketSize : linux 指定要发送数据的字节数。
    • -S hostname/IP addr : linux 将 IP 地址用作发出的 ping 信息包中的源地址。在具有不止一个 IP 地址的主机上，可以使用 -S 标志来强制源地址为除了软件包在其上发送的接口的 IP 地址外的任何地址。如果 IP 地址不是以下机器接口地址之一，则返回错误并且不进行任何发送。
ping应用

ping命令可以用来排除网络连通性的故障，但也有其局限性。下面介绍ping的几种应用场景。

  1. 检测TCP/IP协议是否正常/机器是否可以连通

    利用ping测试本地循环地址/对方机器地址，如果发现本地地址/对方机器地址无法利用ping命令连通，则说明本地TCP/IP协议不能正常工作/机器不能连通。

  2. 判断一条链路的质量

    可以利用它的一些返回数据，来估算你跟某台主机之间的速度是多少字节每秒。

    如上面图片展示的，我们可以看到数据包大小”字节”，发送包的数量 “已发送=4”，接收包的数量”已接收=4”，丢失包的数量”丢失=0”。

    还有往返行程的估计时间Minmum 最小值 、MAXimun 最大值、Average 平均值。

ping返回消息分析

Ping的返回信息有Request Timed Out、Destination Net Unreachable、Bad IP address及Source quench received。

  • Request Timed Out：表示对方主机可以到达到TIME OUT，这种情况通常是为对方拒绝接收你发给它的数据包造成数据包丢失。大多数的原因可能是对方装有防火墙或已下线。

  • Destination Net Unreachable：表示对方主机不存在或者没有跟对方建立连接。

    destination host unreachable和time out的区别

    如果所经过的路由器的路由表中具有到达目标的路由，而目标因为其它原因不可到达，这时候会出现time out，如果路由表中连到达目标的路由都没有，那就会出现destination host unreachable。

  • Bad IP address：表示你可能没有连接到DNS服务器所以无法解析这个IP地址，也可能是IP地址不存在。

  • Source quench received：信息比较特殊，它出现的机率很少。它表示对方或中途的服务器繁忙无法回应。
