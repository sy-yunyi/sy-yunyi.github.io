CSE-CIC-IDS2018 on AWS

通信安全机构(the Communications Security Establishment, CSE)与加拿大网络安全研究所(the Canadian Institute for Cybersecurity, CIC)合作项目

原文地址：https://www.unb.ca/cic/datasets/ids-2018.html

异常检测因其具有检测新攻击的潜力，一直是许多研究者关注的焦点。然而，由于这些系统在部署之前需要进行大量的测试、评估和调优，因此它在实际应用程序中的应用受到了系统复杂性的限制。将这些系统运行在具有全面而广泛的入侵和异常行为集的真实标记网络跟踪上是最理想的测试和评估方法。

这本身就是一个很大的挑战,因为数据的可用性是极其罕见的,因为一方面,许多这样的数据集内部由于隐私问题不能共享,另一方面一些数据集被过度的脱敏,不能反映当前的趋势,或者缺乏一定的统计特征,所以一个完美的数据不存在。所以研究人员通常使用并不是最好的数据集。随着网络行为和模式的变化和入侵的发展，从静态和一次性的数据集转向更动态的生成数据集变得非常必要，这些数据集不仅反映了当时的流量组合和入侵，而且是可修改的、可扩展的和可重现的

为了克服这些缺点，设计了一种系统的方法来生成数据集用来分析、测试和评估入侵检测系统，主要的关注点是基于网络的异常检测器。本项目的主要目标是开发一种系统的方法，在创建包含网络上所见事件和行为的抽象表示的用户概要的基础上，生成用于入侵检测的多样化和全面的基准数据集。这些概要文件将组合在一起，生成一组不同的数据集，每个数据集都具有一组独特的特性，这些特性涵盖了评估域的一部分。

最后的数据集包括七个不同的攻击场景：Brute-force, Heartbleed, Botnet, DoS, DDoS, Web attacks, and infiltration of the network from inside。攻击基础设施包括50台机器，受害者组织有5个部门，包括420台机器和30台服务器。数据集包括每个机器捕获的网络流量和系统日志，以及使用CICFlowMeter-V3从捕获的流量中提取的80个特征。

引文

在CSE-CIC-IDS2018 dataset中，我们使用profile的概念以系统的方式生成数据集，其中将包含应用程序、协议或较低层网络实体的入侵详细描述和抽象分布模型。这些profile 可以被代理或者操作员生成网络中的事件。由于生成的profile的抽象性质，我们可以利用它们在不同的网络拓扑中生成针对不同的网络协议生成事件。同时也可以被用作生成特定需求的数据集。我创建了两类profile:

  • B-profiles：使用各种机器学习和统计分析技术(如K-Means、Random Forest、SVM和J48)封装用户的实体行为。封装的特性包括协议的包大小、每个流的包数、有效负载中的某些模式、有效负载的大小和协议的请求时间分布。在我们的测试环境中，将模拟以下协议:HTTPS、HTTP、SMTP、POP3、IMAP、SSH和FTP。根据我们最初的观察，大部分流量是HTTP和HTTPS。
  • M-Profiles：尝试以明确的方式描述攻击场景。在最简单的情况下，人类可以解释这些概要文件（profile），然后使用它们。理想情况下，将使用自治代理和编译器来解释和执行这些场景。对于攻击，我们考虑了六种不同的场景(表1)：
    • 内网渗透：在此场景中，我们通过电子邮件向受害者发送恶意文件，并利用应用程序漏洞。成功利用后，在受害者的计算机上执行一个后门，然后我们使用他的计算机扫描内部网络，寻找其他易受攻击的盒子，如果可能的话利用它们。
    • HTTP拒绝服务：在这个场景中，我们使用Slowloris和LOIC作为主要工具，这些工具已经被证明可以使用一台攻击机器使得Web服务器完全不可访问。Slowloris从建立到远程服务器的完整TCP连接开始。该工具通过定期向服务器发送有效的、不完整的HTTP请求来保持连接打开，以防止套接字关闭。由于任何Web服务器提供连接的能力都是有限的，所以使用完所有套接字并不能建立其他连接只是时间问题。此外，HOIC【原文中存在笔误】是另一个著名的应用程序，它可以对网站发起DoS攻击。
    • WEB应用程序攻击：在这个场景中，我们使用了Damn Vulnerable Web App (DVWA)作为我们的受害者Web应用程序，它的开发目的是帮助安全专业人员测试他们的技能。在第一步中，我们通过web应用程序漏洞扫描器扫描网站，然后对易受攻击的网站进行不同类型的web攻击，包括SQL注入、命令注入和无限制的文件上传。
    • 暴力攻击：蛮力攻击是非常常见的对网络的攻击方式，因为他们往往盗取弱用户名和密码组合的帐户。最后一个场景的设计目标是通过对主服务器运行字典暴力攻击来获取SSH和MySQL帐户。
    • 最新的漏洞攻击：有一些攻击基于一些著名的漏洞，可以在特定的时间内执行，这些异常的漏洞有时会影响数以百万计的服务器或受害者，通常需要几个月的时间来修补全世界所有易受攻击的计算机，近年来最著名的是心脏滴血。

基础设施和实现

B-Profile

为了产生正常的背景流量，B-Profile被设计用来提取一组人类用户的抽象行为。它试图用机器学习和统计分析技术来封装用户生成的网络事件。封装的特性包括协议的包大小分布、每个流的包数、有效负载中的某些模式、有效负载大小和协议的请求时间分布。一旦从用户抽象出B-profile，代理(CIC-BenignGenerator)或人工操作员就可以使用它们在网络上生成真实的正常事件。相关组织和研究人员可以使用这种方法轻松地生成真实的数据集，因此，不再需要匿名数据集。

M-Profile

我们实现了七个攻击场景。对于每种攻击，我们都基于实现的网络拓扑定义一个场景，并在目标网络之外的一台或多台机器上执行攻击。图1显示了实现的网络，这是AWS计算平台上常见的LAN网络拓扑。为了拥有与真实网络相似的多种机器，我们建立了5个子网络，分别是研发部(Dep1)、管理部(Dep2)、技术部(Dep3)、秘书和运营部(Dep4)、IT部(Dep5)和服务器。除了IT部门，我们为所有部门安装了不同的操作系统 (Windows 8.1和Windows 10)， IT部门的所有计算机都是Ubuntu。对于服务器，我们采用了不同的 Windows服务器，如2012和2016。本节的其余部分将介绍七种攻击场景和工具。

暴力攻击

有很多工具可以进行强力攻击和密码破解，比如Hydra、Medusa、Ncrack、Metasploit模块和Nmap NSE脚本。此外，还有一些用于密码散列破解的工具，如hashcat和hashpump。但是，最全面的多线程工具之一是Patator，它是用Python编写的，似乎比其他工具更可靠、更灵活。它还可以将每个响应保存在一个单独的日志文件中，供以后查看。在这个数据集中，我们使用两个模块，攻击者是Kali Linux机器上的FTP和SSH，受害者是Ubuntu 14.0系统。对于密码列表，我们使用一个包含9000万个单词的大字典。

心脏滴血

利用心脏滴血最著名的工具之一是Heartleech。它可以扫描易受漏洞攻击的系统，然后可以利用它们并泄漏数据。一些重要的特点:

  • 关于目标是否脆弱的结论性/非结论性判断
  • 批量/快速下载heartbleed数据到一个大文件中，使用多个线程进行离线处理
  • 自动检索私钥，不需要其他步骤
  • 一些有限的IDS规避
  • STARTTLS支持
  • IPV6支持
  • Tor / Socks5n代理支持
  • 大量连接诊断信息

为了利用这个漏洞，我们编译了OpenSSL version 1.0.1f，这是一个存在漏洞的版本。然后使用Heartleech检索服务器的内存。

僵尸网络

在这个数据集中，我们使用Zeus，它是一个特洛伊木马恶意软件包，运行在Microsoft Windows的不同版本上。虽然它可以用来执行许多恶意和犯罪的任务，但它经常被用来通过人在浏览器中的击键记录和表单抓取来窃取银行信息。它也被用来安装密码勒索软件。Zeus主要通过下载和网络钓鱼的方式传播。此外，作为补充，我们使用Ares僵尸网络，这是一个开源僵尸网络，具有以下特征：

  • 远程cmd shell
  • 持久性
  • 文件上传/下载
  • 屏幕截图
  • 按键记录

在这个场景中，我们用两个不同的僵尸网络(Zeus和Ares)感染机器，并且每隔400秒我们就向僵尸请求屏幕截图。

拒绝服务

Slowloris是Robert Hansen发明的一种拒绝服务攻击工具，它允许一台机器以最小的带宽和对不相关的服务和端口的副作用来关闭另一台机器的web服务器。在这个场景中，我们使用一个基于perl的Slowloris工具来攻击web服务器。

分布式拒绝服务

High Orbit Ion Cannon，通常缩写为HOIC，是一个开源的网络压力测试和拒绝服务攻击应用程序，使用BASIC编写，可以同时攻击多达256个url。它被设计用来取代普拉托克斯技术公司开发的the Low Orbit Ion Cannon。在这种情况下，我们使用免费的HOIC工具，使用4台不同的计算机进行DDoS攻击。

web攻击

在这项工作中，我们使用脆弱的网络应用程序(DVWA)进行我们的攻击。DVWA是一个脆弱的PHP/MySQL web应用程序。它的主要目标是帮助安全专业人员在法律环境中测试他们的技能和工具，帮助web开发人员更好地理解保护web应用程序的过程，并帮助教师/学生在教室环境中教授/学习web应用程序安全性。为了自动化XSS和蛮力中的部分攻击，我们开发了一个基于Selenium框架的自动化代码。

内网渗透

在这个场景中，应该利用一个脆弱的应用程序，例如Adobe Acrobat Reader 9。首先，受害者通过电子邮件收到恶意文件。然后，在成功利用Metasploit框架后，将在受害者的计算机上执行一个后门。现在我们可以使用Nmap对受害者的网络进行不同的攻击，包括IP扫描、全端口扫描和服务枚举。

特征抽取

CICFlowMeter 是一个用Java编写的网络流量生成器，在选择要计算的特性、添加新特性以及更好地控制数据流超时时间方面提供了更大的灵活性。它产生双向流(Biflow)，其中第一个数据包确定正向(源到目的地)和反向(目的地到源)的方向，因此在正向和反向分别计算了持续时间、数据包数量、字节数、数据包长度等83个统计特征。

应用程序的输出是CSV文件格式，为每个流标记了6列，分别是FlowID、SourceIP、DestinationIP、SourcePort、DestinationPort和具有80多个网络流量特性的协议。通常TCP流在连接断开时终止(通过FIN包)，UDP流通过流超时终止。流超时值可以由单独的方案任意分配，例如TCP和UDP都是600秒。CICFlowMeter-V3提取了80多个特征，如下表所示:

在提取特性并创建CSV文件之后，现在我们需要标记数据。在这里，我们使用攻击场景调度、源和目标的ip和端口以及协议名称来标记每个流的数据。

数据集的使用

数据集是按天被收集。每天，我们记录原始数据，包括每台机器的网络流量(Pcaps)和事件日志(windows和Ubuntu事件日志)。在对原始数据进行特征提取的过程中，我们使用了CICFlowMeter-V3，提取了80多个流量特征，并将其保存到每台机器的CSV文件中。

如果您想使用人工智能技术进行分析，您可以下载我们生成的数据(CSV)文件并分析网络流量。

如果您想使用一个新的特性提取器，您可以使用原始捕获的文件(PCAP和日志)来提取您的特性。然后，您可以使用数据挖掘技术来分析生成的数据。
