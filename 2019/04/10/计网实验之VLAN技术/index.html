计网实验之VLAN技术

Six's Blog
2019-04-10
手稿
VLAN, 计算机网络实验

基本概念

虚拟局域网（Virtual Local Area Network,VLAN）是一种通过将局域网内的设备逻辑的划分成一个个网段，从而实现虚拟工作组的技术。VLAN很好的解决传统网络存在的网络结构、安全性、冗余路径等诸多问题。

利用交换机可以实现VLAN，但并不是所有交换机都具有VLAN功能。

一个VLAN组成一个逻辑子网，即一个逻辑广播域，可以覆盖多个网络设备，允许处于不同网络位置的网络用户加入到同一个逻辑子网中。IEEE与1999年颁布了用以标准化VLAN实现方案的IEEE 802.1q协议标准草案。

VLAN是为来解决以太网的广播问题和安全性问题而提出的一种协议，它在以太网帧的基础上增加了VLAN首部，用VLAN ID 把用户划分为更小的工作组，限制不同工作组之间的二层访问，每个工作组就形成了一个虚拟的局域网。

虚拟局域网的主要特点是可以限制广播范围，形成虚拟工作组，动态管理网络。

每个VLAN都包含一组有相同需求的工作站，与物理上形成的LAN有相同的属性。但由于其是逻辑上划分而不是物理划分，所以这些工作站不一定属于同一个物理网段。

一个VLAN内部的广播和单播流量都不会转发到其他VLAN中，从而有助于控制流量、减少设备投资、简化网络管理和提高网络的安全性。

VLAN的特征：

  • 不受网络物理位置的限制，可跨越多个物理网络和多台交换机。

  • 可隔离广播信息，每个VLAN为一个广播域，可以有效防止广播风暴的发生。

    如果要实现不同VLAN之间的主机通信，则必须通过一台路由器或者三层交换机实现。

  • 可有效提升带宽，将网络上的用户按业务功能划分多个逻辑工作组，那么大部分通信交流将被限制在一个VLAN中，使带宽得到有效提升。

  • VLAN由软件实现定义和划分，建立和重组灵活。

VLAN分类

  1. 基于端口划分VLAN - 静态VLAN

    优点在于在定义VLAN成员时非常简单，只需将所有端口都定义为相应的VLAN即可，且适用于任何大小的网络。例如可以指定交换机1的端口1~ 6和交换机2的端口1~4端口为同一VLAN，即实现了跨越多台交换机的VLAN。

    缺点在于用户从一个端口迁移至另一个端口时需要重新定义。

  2. 基于MAC地址划分VLAN - 动态VLAN

    利用MAC地址区分不用的用户，根据规则将不同的用户划分至不同的VLAN。

    优点在于用户物理移动不用重新配置VLAN。

    缺点是初始化时，所有用户都必须进行配置，如果用户较多，则配置繁琐，适用于小型的局域网。

  3. 基于网络层协议划分VLAN - 动态VLAN

    可分为IP、IPX、AppleTalk等，根据不同的协议类型进行流量控制。

    优点是用户的物理位置改变时，既不需要重新配置所属VLAN，也不需要附件帧标签，可以减少网络的通信量。

    缺点是效率低，检查IP帧头增加了延时时间。

  4. 根据IP组播划分VLAN - 动态VLAN

    此方法将一个IP组播组划分为一个VLAN，主要适用于将不在同一地理范围内的局域网组成一个VLAN。

VLAN数据帧标识

最具代表性的VLAN数据帧标识方法时虚拟桥接局域网标准IEEE 802.1q。此协议定义了同一VLAN跨越交换机通信桥接的规则以及正确的标识VLAN的帧格式。IEEE 802.1q标准使用一种内部标记tag机制，该机制将标记插入到帧内，重新计算帧校验序列。每个标记包含4B，其中VPID是2B的控制信息，包含3位用户优先级，1位规范格式指示（默认为0，表示以太网）和12位的VLAN标识符。

基于IEEE 802.1q标准的tagVLAN用VID划分不同的VLAN，当数据帧通过交换机时，交换机根据数据帧中的tag的VID信息，标识特面所在的VLAN，这使得所有属于该VLAN的数据帧，都被限制在该逻辑VLAN中传输。

数据帧的格式如下：

前导帧 隔符 目的MAC地址 源MAC地址 类型 数据部分 帧检测序列
7B 1B 6B 6B 2B 46~1500B 4B

采用IEEE 802.1q方法标识的数据帧格式如下：

前导帧 隔符 目的MAC地址 源MAC地址 VPID VCI 类型 数据部分 帧检测序列
7B 1B 6B 6B 2B 2B 2B 46~1500B 4B

以太网数据帧最大长度为1518B，加上VPID和VCI字段后长度增加至1522B。

VLAN中的端口

VLAN以VLAN ID 标识，ID 的取值范围为1~4096。一台交换机最多支持250个VLAN，其中，VLAN 1 是出厂默认设置的VLAN，如果不另行配置，所有交换机连接的设备都是VLAN1。VLAN1是不可删除的VLAN。

VLAN端口分为两种：Access端口和Trunk端口。

Access端口只能属于一个VLAN，并且可以通过手工设置指定VLAN。此端口既不能直接从另一个VLAN接收信息，也不能像其他VLAN发送信息，只能传输属于此VLAN的帧。

Trunk端口默认属于本交换机所有VLAN，通常位于交换机之间的来连接，以保证在跨越多台交换机上建立的同一个VLAN的成员可以通信。交换机之间互连用的即为Trunk端口，可以承载带VLAN和不带VLAN的报文。

交换机的Trunk端口不属于某个VLAN，而是承载所有VLAN的帧，帧的格式是IEEE 802.1q。Trunk端口基于OSI第二层。
