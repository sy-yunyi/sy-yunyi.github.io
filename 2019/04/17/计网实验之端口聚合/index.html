端口聚合

端口聚合(Aggregating Port,AP)又称端口绑定，链路聚合，遵循IEEE 802.3ad协议的标准。

端口聚合是指将多个端口聚合在一起形成一个聚合组，以实现将输入\输出的负荷分担在各成员端口中，同时也提供了更高的连接可靠性。

端口聚合服务的上层实体将同一聚合组内的多条物理链路是为一条逻辑链路，从整体上看，一个聚合组形同一个端口。

端口聚合是在MAC子层和物理层之间的小层次，即数据链路层和物理层之间。

为了不改变以太网的协议栈，将链路聚合归结为数据链路层，因此端口聚合也成为链路聚合。

端口聚合的特点

  • 实现了各个聚合端口的符合分担及同组成员之间的动态备份

  • 增加了链路带宽，又防止形成环路

  • 聚合后的带宽是每个独立端口的带宽总和

    当端口上流量增加而成为限制网络性能的瓶颈时，可以通过端口聚合增加带宽

  • 支持流量平衡，可以把流量均匀的分配至各成员链路

  • 链路备份，当聚合组中一条成员链路断开时，将其链路上的流量自动分配至其他成员链路上

端口聚合限制条件

  1. 只有同类型端口且双工速率一致才能聚合为一个AG端口。光口和电口不能绑定。

    AP成员端口速率要一致

    端口类型要一致，二层端口只能加入二层AP；三层端口只能加入三层AP

  2. 所有物理端口必须属于同一个VLAN。

  3. 最多支持8个物理端口聚合为一个AG。

  4. 当一个端口加入AP后,不能在该端口上进行任何配置,直到该端口退出AP。

  5. AP不能设置端口安全功能。

  6. 若端口加入AP，则端口的属性被AP属性取代。

  7. 若端口从AP中删除，则端口属性恢复至加入AP前的属性。

配置流量平衡

流量平衡又称负载均衡。其建立在现有网络上，通过一种廉价、有效、透明的方法扩展网络设备和服务器的带宽并增加吞吐量，加强网络数据处理能力，提高网络的灵活性和可用性。

流量平衡

  • 将大量并发访问或者数据流量分担到多台节点设别分别处理，减少用户的响应时间
  • 将单个重负载的运算分担到多个节点设备做并行处理，系统处理能力得到提高

流量平衡模式

端口聚合的流量平衡模式有很多，包括根据报文的源 MAC 地址、目的 MAC 地址、源 MAC 地址+目的 MAC 地址、源 IP 地址，目的 IP 地址以及源 IP 地址+目的 IP 地址等特征值把流量平均地分配到 AP 的成员链路中。

源MAC地址流量平衡是根据报文的源MAC地址把报文分配至AP的各个成员链路中。不同源MAC地址的报文，其转发链路不同，相同的源MAC地址报文向同一个成员链路转发。

目的MAC地址流量平衡是根据报文的目的MAC地址把报文分配至AP的各个成员链路中。相同源目的MAC地址的报文同一个成员链路转发，不同的源MAC地址报文转发链路不同。

源MAC+目的MAC流量平衡是根据报文的源MAC地址和目的MAC地址把报文分配至AP的各个成员链路中。不同MAC地址的报文，其转发链路不同，相同的MAC地址报文向同一个成员链路转发。

源IP地址或目的IP地址流量平衡是根据报文的源IP地址和目的IP地址进行流量分配。该模式用于三层报文，如果该模式下收到二层报文，则根据报文中的源MAC地址或者目的MAC地址进行转发。

源IP地址+目的IP地址流量平衡是根据报文的源IP地址和目的IP地址进行流量分配。该模式用于三层报文，如果该模式下收到二层报文，则根据报文中的源MAC地址或者目的MAC地址进行转发。具有不同的源IP地址+目的IP地址的报文可能会被分配至同一个成员链路中。
