交换机, 计算机网络实验

交换机工作在OSI模型的第二层（数据链路层），又称二层交换机。其基于MAC地址识别完成数据帧封装、转发。

交换机可以在传统的LAN中消除竞争和冲突，隔离广播域。

以太网交换机类似于一台专用的计算机，由中央处理器、随即存储器和接口组成，适用于连接工作站、服务器、路由器和其他交换机，主要作用是快速高效、准确无误地转发数据。

交换机工作原理

交换机检查收到的数据帧中的目的端口，根据端口决定数据帧的流向。交换机通过自学习形成一个MAC表，根据MAC表实现对数据帧的过滤和转发，降低错误数据帧的发生概率。

两个基本操作：

  • 交换数据帧，将从某一端口收到的数据帧发到该帧的目的端口
  • 维护交换操作，构造和维护动态MAC表

交换数据帧

交换机接收到数据帧后，先将其串行代码转化为并行代码，存储在快速缓存RAM中，进行查表操作，确定目标端口，交换机在源端口与目的端口之间建立虚连接，然后将并行代码转化为串行代码，发送至目的端口。

构造和维护MAC表

MAC地址表中，每一条表项存储着一个连接在交换机端口上的设备的MAC地址及其相应的端口。

MAC地址表的建立和维护过程：

  1. 收到数据帧，检查该帧的源和目的MAC地址，与内部MAC表进行比较；
  2. 若数据帧源MAC地址不在表中，将该源MAC地址及其对应的端口号加入MAC表中；
  3. 若目的MAC地址在该表中，则将数据帧发送至相应的目的端口；
  4. 若不在，则将该数据帧发送至所有其他端口；

当计算机的交换机加电、断电或迁移时，网络的拓扑结构会发生改变。为了适应的动态拓扑的问题，MAC表项中含有帧到达的时间，通过时间对比记录到达该交换机的最新的帧。

由于交换机中的内存有限，能够存储的MAC地址数有限，所以交换机也会设定一个自动老化时间，若某条记录在设定的时间段内没有得到更新则将其清除。

交换机可以进行全双工传输，可以同时在多对节点之间建立临时专用通道，形成立体交叉的数据传输通道结构。

交换机地址表的大小会影响交换机的接入容量。

基本功能

  • 地址学习

  • 转发/过滤决策

    • 若数据帧的目的MAC地址是广播地址或组播地址，则交换机向除源端口外所有端口转发
    • 若为单播地址，但该地址不在表中，也向除源端口外所有端口转发
    • 若地址在表中，则转发至指定端口
    • 如果数据帧的目的地址与源地址在同一个物理网段，则丢弃数据帧，不做交换

  • 避免环路

    生成树协议

性能指标

端口数量、背板带宽、延时、扩展性、机架插槽数与扩展槽数、三层交换技术、芯片、管理功能、单/多MAC地址类型、光纤解决方案

交换方式

直通方式

直通方式（Cut-through）指在输入输出端口检测到数据帧在读取目的地址后，立刻查表，并在输入/输出交叉处接通，将数据帧直通到相应的端口，实现交换功能且无需读物帧的全部内容。

优点：无需存储，延时时间短，交换速度快

缺点：无错误检测能力；不能将不同的输入/输出端口直接连通；容易丢帧

存储转发

存储转发（Store&Forword）是将数据帧完整的存入缓存，然后进行CRC检查，对错误帧处理后再通过查表找出目的端口进行交换。

优点：正常帧可以通过，错误帧被过滤；通过对错误帧的处理，改善网络性能；实现高速端口与低速端口的协同工作

缺点：数据处理时延长

碎片隔离

碎片隔离（Fragment Free）是一种介于前两者之间的解决方案，速度介于两者之间，也称为无碎片直通转发。检查数据帧的大小是否大于64B，如果是则认为是正常数据帧，进行转发；否则认为是帧碎片，丢弃。但对正常帧不进行数据校验，不具有错误检测能力。

优点：交换速度较快，可以有效降低错误帧的转发

缺点：无数据校验能力；无法识别大于64B的帧碎片；转发延时大于直通

配置基础

交换机不进行配置直接使用就可以实现其基本功能：数据交换。

通过配置管理们可以实现更多的功能。

交换机的管理方式分为两种

  • 带外管理

  • 带内管理

    Telnet远程管理

    Web远程管理

    SNMP工作站远程管理

首次配置交换机只能使用带外管理。

带外管理

带外管理方式（Out-band Managenment）一般利用Windows自带的超级终端程序完成。

  1. 利用线缆将交换机与计算机相连。
  2. 运行“超级终端”，若没有则需要手动安装。
  3. 进行相关配置的设置。

win的超级终端每次使用都要进行配置，使用不方便，并且目前已经不再支持。终端仿真程序VanDyke SecureCRT 更加实用。

带内管理

Telnet管理

Telnet协议是一种远程访问协议，通过它可以登陆到远程计算机、网络设备或专用TCP/IP网络。

Web管理

Web管理的条件:

  • 交换机支持HTTP
  • 配置管理VLAN的IP
  • 交换机有独立IP
  • web主机与交换连通
  • web主机所在VLAN为管理VLAN
SNMP管理
  • 交换机支持SNMP或RMON管理协议，支持网管软件
  • 交换机配置了管理VLAN的IP
  • SNMP主机与交换机的IP在同一网段
  • SNMP主机所在VLAN为管理VLAN

命令模式

模式是针对不同的配置要求。

交换机一旦与网络设备建立连接，则进入普通用户模式。这种模式下只能查看网络设备的运行状态和简单的统计信息。

其他操作则要进入特权模式，需要管理账号和密码。

在特权模式下可以通过命令configure terminal进入全局配置模式。

不同的模式对应不同的命令集，只有进入对应的模式才能执行相应的配置。

用户模式分类

  1. 用户模式

    提示符：>

    此模式下能进行基本的测试和显示系统信息

  2. 特权模式

    提示符：#

    验证设置命令的结果。该模式可设置口令保护。

    在用户模式下使用enable命令进入特权模式

  3. 全局配置模式

    提示符：(config)#

    配置影响整台交换机的全局参数。

    在特权模式下使用configure terminal命令进入全局配置模式

  4. 接口配置模式

    提示符：(config-if)#

    配置交换机的各种接口。

    在全局配置模式下使用interface命令进入接口配置模式

  5. VLAN配置模式

    提示符：(config-vlan)#

    配置VLAN参数

    在全局配置模式下使用vlan vlan-id进入VLAN配置模式
