Security of Software Defined Networks: Taxonomic Modeling, Key Components and Open research area

来源

Kaur R, Singh A, Singh S, et al. Security of software defined networks: Taxonomic modeling, key components and open research area[C]//Electrical, Electronics, and Optimization Techniques (ICEEOT), International Conference on. IEEE, 2016: 2832-2839.

为了做关于SDN安全的报告而读的一篇论文，可能看的并不仔细。

软件定义网络

软件定义网络承诺网络运营商大大简化网络管理，其通过网络可编程性提供灵活性和创新。 借助SDN，网络管理从低级设备配置的编码功能转变为构建便于网络管理和调试的软件，为解决存在已久的问题提供了一种新的方法。

但是，SDN的发展也带来了新的挑战。

各种架构组件的引入和SDN新颖实体的上升循环带来了新的安全问题和威胁。

SDN被认为是数字威胁和网络攻击的主要目标，并且比简单网络具有更具破坏性的影响。

SDN的架构引发了新的外部和内部威胁和漏洞。

  • 完整性和安全性没有被证明，更容易被通过单点攻击整个网络
  • 可编程性为攻击者提供了执行恶意代码的能力
  • 南向接口可能存在拒绝服务攻击和侧信道攻击
  • 配置错误可能带来更大的危害。
  • SDN的代理可能被注入错误的流信息。

可能漏洞

  • 认证攻击
  • 数据泄露
  • 恶意注入攻击
  • 控制器劫持或欺骗规则注入
  • 拒绝服务攻击
    • 控制器-交换器通信洪泛
    • 交换器流表洪泛
  • 配置错误

安全解决方案分类

  1. Secure Composition
    • FRESCO [6] - 主要解决安全可编程问题，对于除了数据层其他层都有很大的影响，它旨在解决可加速设计新的SDN安全服务的各种关键问题。它还导出一个脚本API，使安全专家能够将安全监控和威胁检测逻辑编码为模块化库。
    • FORTNOX [2] - 新的安全策略实施内核。它处理授权和规则冲突，并对接口和控制层产生直接影响。
  2. Security Audit
    • VERIFICARE - 给出满足设计要求的模型。该提议作为验证网络正确性和规范建模的验证平台。
    • SDN DEBUGGER - SDN调试器帮助研究人员找出bug的根本原因。它捕获并重建导致错误行为的事件序列。调试器可帮助研究人员解决逻辑错误，并帮助网络运营商收集错误报告，然后将其提交给供应商。
  3. Security enforcement policy
    • VERIFLOW - Veriflow [6]基本上实时验证了网络范围内的不变量。 它是SDN控制器和网络设备之间的一个层，可以在插入修改和删除的每个转发规则时动态检查实时不变量违规。
    • FLOVER - Son.et al提出的Flover [6]基本上是一种检查模型，用于根据网络安全策略确定和验证流策略。它有助于设计复杂的威胁检测和缓解模块。
    • Perm OF - 一个细粒度的权限系统，包含各种特定于OF的权限。
  4. Security Enhancement
    • FleXam - FleXam是OpenFlow的采样扩展，允许Controller定义应该采样的数据包，应该选择每个数据包的哪些部分以及应该将它们发送到何处。FleXam使控制器能够动态调整采样率，从而降低控制平面负载。
    • CLOUDWATCHER - 一种监控云的新安全监控服务模型。CLOUDWATCHER [7]改变了网络流量的路由路径，它使流量通过安全设备所在的网络节点进行传输。
    • L-IDS - 学习入侵检测系统（IDS）。L-IDS是移动设备的安全服务，用于在特定区域或位置进行保护。
  5. Security Analysis
    • OpenWatch - 它是一个程序，可以将不同的异常检测应用程序作为输入，并决定应监视哪些流程
    • AVANT-GUARD - 包含两种技术，第一种技术是连接迁移，它显着减少了由于南向接口上的DOS攻击而产生的数据到控制平面的交互。另一种技术是通过在SDN设备中提供条件触发推送功能来解决响应性挑战的致动触发器。
    • HEADER SPACE ANALYSIS - 通过检查防火墙流量空间和授权空间来解决SDN防火墙中的问题和冲突。此方法的功能是搜索整个网络中的流路径，并根据所有防火墙拒绝规则确定流路径以确定问题。

SDN安全的关键组件

  1. Securing the control layer - 控制器被称为SDN的大脑或所有功能发生的中央决策点。作为一个中心点，控制器是黑客感兴趣的攻击目标。潜在的攻击者可以尝试控制网络，伪装成SDN控制器或者只是闯入控制器本身。为了保护SDN控制器，需要强大的访问控制，需要DDoS保护，防病毒和其他威胁防御和检测，缓解技术。
  2. Securing traffic flow - SDN基于智能转发流量。它是SDN功能的主要部分，因此必须加以保护。

SDN安全的开放研究领域

  1. Operating system alteration- 操作系统是管理网络硬件和软件资源并为网络程序提供公共服务的软件。 如果操作系统发生变化，结果将足够危险。它可以感染所有层和接口，并可能影响应用程序管理功能。操作系统的更改可以通过可信计算进行保护，但是需要在此领域开展工作的研究。
  2. Unauthorized Intrusion - 这是一种非法访问任何网络的行为。所有层和接口都是这些类型攻击的目标。它可能破坏网络的所有功能。所以，需要安全管理模块作为保护技术。
  3. Several attacks - 对于SDN安全性，应该仔细确定几种类型的攻击。大多数攻击都是由研究人员解决的，但是还存在其他一些攻击仍未得到解决，如欺骗攻击。
