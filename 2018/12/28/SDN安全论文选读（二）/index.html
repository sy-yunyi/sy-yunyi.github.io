SDN安全论文选读（二）

发表于 2018-12-28 | 分类于 论文选读

Comparative Analysis of Control Plane Security of SDN and Conventional Networks

来源

Abdou A R, van Oorschot P C, Wan T. Comparative Analysis of Control Plane Security of SDN and Conventional Networks[J]. IEEE Communications Surveys & Tutorials, 2018.

背景介绍

网络设备的三种类型：

  • pure SDN - no control function and is fully controlled by an external SDN controller
  • pure CN - implements all of its own control functions and is not controlled by any SDN controller
  • hybrid - both implements control functions, and can be controlled by an SDN controller

潜规则，在学术界，讲SDN为pure SDN，在工业界为 hybrid networks 。

在实际应用中，更关注的是SDN的可编程能力，而不是控制与数据的分离。

学术界对传统网络的控制方法的安全研究不足，虽然SDN的控制层安全收到了比较多的关注，但是在实际的网络中仍然没有得到实现。

本文提供了一个框架，其中包含生产网络所需的基本网络属性。 使用它，我们分别研究SDN和CN如何实现这些属性，并相应地分析安全攻击和缓解。

Conventional Networks

CN可以是L2或L3。 仅由L2交换机组成的网络作为其中间系统称为L2网络。可以使用例如L3路由器连接两个（或更多个）L2网络。 L3路由器网络称为L3网络。

L2或L3的不同：

  • different mechanisms in constructing their forwarding tables.
  • handle unknown packets differently.
网络属性

  • Basic Forwarding 基础转发

  • Loop Free Forwarding 无环路转发

  • Link Redundancy 链接冗余

    在两个网络设备间存在多个链路，若一个或多个失效，则网络的拓扑是不变的。并且可以通过多条链路同时传输数据。

  • Device Redundancy 设备冗余

    高可用性。 如果任何一个设备发生故障，由两个或多个设备组成的网络应保持完全可用。

  • Scalability 可扩展性

    网络设计应该允许增长而无需显着的管理开销

SDN威胁分析

  1. 基础转发

    • Host Profile Poisoning 主机配置文件污染

      类型：MAC spoofing, IP spoofing, VLAN tag spoofing

      原因：dishonest host or switch can forge such information inside a packet

    • flooding attacks 洪泛攻击

      类型：DoS or packet interception

      原因：generate a large number of packets with arbitrary MAC and IP addresses

    • 防御措施

      MAC spoofing ：MAC binding

      VLAN tag spoofing ：指定VLAN 端口，remove VLAN tags

      IP spoofing：规划IP地址分配和配置带有IP前缀的流表，避免从数据平面学习基于IP的转发规则

  2. Loop Free Forwarding

    • 链路层发现协议(Link Layer Discovery Protocol,LLDP) spoofing attacks

      类型：Link Fabrication Attack ，LLDP flooding

      原因：攻击者可以发送虚假的LLDP包给控制器，污染网络拓扑的计算。

    • 防御措施

      Message authenticity and integrity

      Consistency checks of LLDP packets

      指定LLDP端口

  3. Link Redundancy

    • message spoofing attacks

      原因：攻击者可以向控制器发送虚假链接上下事件，以操纵链接组的状态。

    • 防御措施

      Message authenticity and integrity

      Consistency checks of OpenFlow events

  4. Device Redundancy

    • spoofing attacks

    • 防御措施

      message origin authentication and integrity

      information corroboration

  5. Scalability

    • 漏洞可能来自对等模型中的分布式通信，也可能来自层次模型中的控制器 - 控制器通信

      如果使用BGP在控制器之间交换信息，可以利用BGP中的漏洞攻击SDN。

    • 如果分布式数据库用于控制器之间的同步，则漏洞也可能来自分布式数据库。

      查询注入攻击

    • 防御措施

      data origin authentication and message integrity

      information corroboration

SDN与传统网络威胁对比

  1. Basic Forwarding

    MAC table poisoning (against a CN switch)

    host profile poisoning (against an SDN controller)

    本质上类似的，但在某些细节上是不同的。

    MAC table poisoning的攻击向量是虚假的MAC地址，而host profile poisoning的攻击向量同时需要MAC和IP地址

    传统网络并不容易受到DOS攻击，这是SDN的一个新的攻击面

  2. Loop Free Forwarding

    传统的二层或三层网络利用STP和路由协议来实现无环转发，而SDN利用LLDP实现拓扑发现。

    攻击者都可以利用协议漏洞来进行攻击，但攻击方式不同。

  3. Link Redundancy

    传统网络利用链路聚合控制协议(Link Aggregation Control Protocol)聚合路由，SDN控制器监听链接的状态变化更新链接集合。

    CN- LACPDU spoofing

    spoofed link-manipulation messages

Control Plane

  • Security-Oriented Controllers

    proposed SE-Floodlight, a security enhanced system based on Floodlight[12]

    • role-based permission model
    • authentication
    • flow conflict resolution services
    • enforce privilege separation

  • Control Plane Security Extensions and APIs

    monitor and detect suspicious network behavior

    • real time checking and verification of forwarding behavior [62]
    • providing logical separation of multiple controller instances [68]
    • Kinetic to not only monitor network properties, but also enable administrators to take
      appropriate control actions in response to network changes, and to analyze source of errors in control programs leading to the undesired network behavior. [73]

  • Attack Mitigation

    machine learning can be used to identify traffic involved in a DoS attack [79]

    proposed as an anti-spoofing mechanism, aiming to mitigate DoS [80]
